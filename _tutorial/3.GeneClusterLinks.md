Plant specialised metabolites are often produced by **biosynthetic gene clusters (BGCs)** — groups of physically co-located genes that together encode a metabolic pathway.

PlantMetWiki provides explicit **cross-links between metabolic pathways and BGC resources**, allowing you to move from:

- pathway-level knowledge
- to genomic context
- to specialised metabolite biosynthesis

In this section, we explore how PlantMetWiki connects pathways to:

- **plantiSMASH** predictions (computationally predicted BGCs)
- **MIBiG** curated BGCs (experimentally validated)
- external metadata describing gene clusters

**SPARQL endpoint**  
https://plantmetwiki.bioinformatics.nl/sparql

## BGC data lives in dedicated named graphs

Unlike pathway content (which lives in `graph/pathways`), BGC data is stored in separate named graphs:

| Graph | Contents |
|---|---|
| `http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-plantismash` | plantiSMASH v2 predictions — 65 BGCs, *Arabidopsis thaliana* only |
| `http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-mibig` | MIBiG 4.0 curated BGCs — 43 BGCs |

BGC nodes are typed `pmw:BiosyntheticGeneCluster`:

```sparql
PREFIX pmw: <http://rdf-plantmetwiki.bioinformatics.nl/vocab/>

?bgc a pmw:BiosyntheticGeneCluster .
```

## How BGC–pathway links work: the shared gene IRI join

BGCs are **not** directly connected to pathway IRIs by a single predicate. Instead, the connection is made through **shared gene IRIs** from identifiers.org:

1. A BGC `ro:0000051` (RO *has_part*) a gene IRI from identifiers.org
2. The same gene IRI appears in `graph/pathways` as a `wp:GeneProduct` that is `dcterms:isPartOf` a pathway

The join therefore spans **three named graphs**:

```
bgc-plantismash (or bgc-mibig) → shared gene IRI → graph/pathways
```

The required prefix for the RO relation:

```sparql
PREFIX ro: <http://purl.obolibrary.org/obo/RO_>
```

The join pattern looks like this:

```sparql
GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-plantismash> {
  ?bgc ro:0000051 ?gene .
}
GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/pathways> {
  ?gene a wp:GeneProduct ;
        dcterms:isPartOf ?pw .
  ?pw dc:title ?title .
}
```

## Listing all plantiSMASH BGC–pathway cross-links

```sparql
PREFIX wp:      <http://vocabularies.wikipathways.org/wp#>
PREFIX dcterms: <http://purl.org/dc/terms/>
PREFIX dc:      <http://purl.org/dc/elements/1.1/>
PREFIX ro:      <http://purl.obolibrary.org/obo/RO_>
PREFIX pmw:     <http://rdf-plantmetwiki.bioinformatics.nl/vocab/>

SELECT DISTINCT ?bgc ?pw (STR(?titleLit) AS ?title)
WHERE {
  GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-plantismash> {
    ?bgc a pmw:BiosyntheticGeneCluster ;
         ro:0000051 ?gene .
  }
  GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/pathways> {
    ?gene a wp:GeneProduct ;
          dcterms:isPartOf ?pw .
    ?pw dc:title ?titleLit .
  }
}
ORDER BY ?title
LIMIT 200
```

plantiSMASH cluster IRIs follow the pattern:

```
https://plantismash.bioinformatics.nl/precalc/v2/<cluster-id>
```

## Combining plantiSMASH and MIBiG in a single query

Some pathways link to both predicted and curated clusters. You can retrieve all BGC–pathway links regardless of source using `UNION`:

```sparql
PREFIX wp:      <http://vocabularies.wikipathways.org/wp#>
PREFIX dcterms: <http://purl.org/dc/terms/>
PREFIX dc:      <http://purl.org/dc/elements/1.1/>
PREFIX ro:      <http://purl.obolibrary.org/obo/RO_>
PREFIX pmw:     <http://rdf-plantmetwiki.bioinformatics.nl/vocab/>

SELECT DISTINCT ?bgc ?source ?pw (STR(?titleLit) AS ?title)
WHERE {
  {
    GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-plantismash> {
      ?bgc a pmw:BiosyntheticGeneCluster ;
           ro:0000051 ?gene .
    }
    BIND("plantiSMASH" AS ?source)
  }
  UNION
  {
    GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-mibig> {
      ?bgc a pmw:BiosyntheticGeneCluster ;
           ro:0000051 ?gene .
    }
    BIND("MIBiG" AS ?source)
  }

  GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/pathways> {
    ?gene a wp:GeneProduct ;
          dcterms:isPartOf ?pw .
    ?pw dc:title ?titleLit .
  }
}
ORDER BY ?source ?title
LIMIT 200
```

This allows you to compare predictions with curated knowledge and identify pathways that have experimental validation.

## Pathway-centric view: which pathway belongs to a specific BGC?

You can start from a specific MIBiG BGC and ask which pathway its genes belong to.

Example: **MIBiG BGC0000670** is the thalianol BGC in *Arabidopsis thaliana*.

```sparql
PREFIX wp:      <http://vocabularies.wikipathways.org/wp#>
PREFIX dcterms: <http://purl.org/dc/terms/>
PREFIX dc:      <http://purl.org/dc/elements/1.1/>
PREFIX ro:      <http://purl.obolibrary.org/obo/RO_>

SELECT DISTINCT ?pw (STR(?titleLit) AS ?title)
WHERE {
  GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-mibig> {
    <https://bioregistry.io/mibig:BGC0000670> ro:0000051 ?gene .
  }
  GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/pathways> {
    ?gene a wp:GeneProduct ;
          dcterms:isPartOf ?pw .
    ?pw dc:title ?titleLit .
  }
}
ORDER BY ?title
```

## BGC-centric view: which BGCs are linked to a specific pathway?

You can also start from a pathway and retrieve all associated BGCs from both sources:

```sparql
PREFIX wp:      <http://vocabularies.wikipathways.org/wp#>
PREFIX dcterms: <http://purl.org/dc/terms/>
PREFIX ro:      <http://purl.obolibrary.org/obo/RO_>
PREFIX pmw:     <http://rdf-plantmetwiki.bioinformatics.nl/vocab/>

SELECT DISTINCT ?bgc ?source
WHERE {
  GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/pathways> {
    ?gene a wp:GeneProduct ;
          dcterms:isPartOf ?pw .
    FILTER(CONTAINS(STR(?pw), '/PC629_'))
  }
  {
    GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-plantismash> {
      ?bgc ro:0000051 ?gene .
    }
    BIND("plantiSMASH" AS ?source)
  }
  UNION
  {
    GRAPH <http://rdf-plantmetwiki.bioinformatics.nl/graph/bgc-mibig> {
      ?bgc ro:0000051 ?gene .
    }
    BIND("MIBiG" AS ?source)
  }
}
ORDER BY ?source
```

## plantiSMASH vs MIBiG: key differences

| Feature | plantiSMASH | MIBiG |
|---|---|---|
| Method | Computational prediction | Manual curation |
| Species coverage | *Arabidopsis thaliana* only (v2) | Multi-species, plant-focused |
| Number of BGCs in PMW | 65 | 43 |
| Evidence level | Predicted | Experimentally validated |
| IRI pattern | `https://plantismash.bioinformatics.nl/precalc/v2/...` | `https://bioregistry.io/mibig:BGC...` |

## Why BGC cross-links matter

By linking pathways to gene clusters through shared gene IRIs, PlantMetWiki enables:
- genome-to-metabolite reasoning
- discovery of candidate biosynthetic loci
- comparison of predicted vs curated clusters
- integration with omics pipelines

This makes PlantMetWiki especially useful for:
- plant specialised metabolism research
- natural product discovery
- functional genomics
- comparative pathway analysis
